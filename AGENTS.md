# Agent Behavior Rules

## General

- Do not generate overly complex solutions
- Always prefer simplicity and clarity
- Avoid unnecessary abstractions

## Code Quality

- Write clean, readable, production-ready code
- Use semantic HTML whenever possible
- Avoid excessive nesting and divs
- Do not generate unused code

## React / Next.js

- Use functional components
- Follow Next.js App Router conventions
- Keep components small and reusable
- Avoid large monolithic components

## Tailwind CSS

- Use Tailwind for styling
- Keep classNames clean and readable
- Avoid duplicated or conflicting styles

## UI Implementation

- Match the Figma design closely
- Respect spacing, typography, and layout
- When unclear, prioritize usability over exact visual match

## AI Assistance

- When generating UI, think in components
- Do not generate full pages unless explicitly asked
- Prefer incremental building (section by section)
